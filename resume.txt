# Changes since v4.5.2

## Improved help documentation for bash helpers (eab660b)

- Enhanced help text for tools/helpers.bash functions

## Configurable docker container name (14c5dfd)

- H2agent runner now accepts a custom container name prefix

## Pipe support for helper functions (51fb8b0)

- `global_variables` and `client_provision` now accept process substitution input

## Consolidated client provision helper and fixed pipe detection (6d7b6ca)

- Removed `launch_client_provision` (redundant, used wrong HTTP method PUT instead of GET)
- Consolidated triggering into `client_provision` with `--in-state <state>` flag
- Fixed process substitution support: `[ -f "$1" ]` → `[ -f "$1" -o -p "$1" ]`

## Fixed doxygen typo and added component test for seq source (dc819a6)

- Fixed `Server` → `Client` in MockClientEvent.hpp doxygen comment
- New component test `test_058_sequenceSourceInClientProvision` validating `seq` source substitution

## Fixed URL query parameters broken by unquoted arguments in do_curl (b6ef2f4)

- `$@` → `"$@"` in `do_curl` to prevent shell from splitting URLs with `&`

## Zero-based sequence and signed integer support for client provisions (e5c2bb9)

- Client provision sequence type changed from `uint64_t` to `int64_t` (±9.2×10¹⁸ range)
- Negative sequence values now allowed for `sequence`, `sequenceBegin`, `sequenceEnd`
- Timer-based triggering fixed: sequence now starts at `sequenceBegin` (0-based) instead of `sequenceBegin+1`
- Affected files: AdminClientProvision, MockClientEvent, MockClientData, MockClientEventsHistory, MyAdminHttp2Server
- Updated: openapi.yaml, API README, unit tests
